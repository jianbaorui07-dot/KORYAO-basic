# 星桥三联：面向中国用户的 Codex 本地创作软件接入方案

> English name: **StarBridge Trinity Protocol**

星桥三联是一套面向中国个人创作者、小团队、设计工作室、工程制图人员和本地 AIGC 工作站的接入方案。它把 Codex、GitHub/Jules、ComfyUI、Blender、CAD、Photoshop 和 Illustrator 放在一条可追踪的工作链里：Codex 负责编写脚本、整理流程、调用本机接口和记录结果，专业软件继续负责图像、三维、制图、修图和矢量设计。

这个项目主要服务于中国用户。文档优先使用中文，示例优先照顾 Windows 本机环境，默认不要求把模型、素材、客户图纸、PSD/AI 工程或本机路径上传到云端。能在本机完成的生成、修图、制图、矢量化和渲染，就优先留在本机完成。

## 面向中国用户的定位

- **中文优先**：说明、索引、任务提示和安全边界优先用中文写清楚，命令、API、workflow、MCP 等必要术语保留英文。
- **本机优先**：适配中国用户常见的 Windows 创作工作站，把 ComfyUI、Adobe 软件、AutoCAD、Blender 等本机软件接进 Codex。
- **资产不外传**：客户图纸、商业素材、模型文件、微信/桌面临时文件、生成输出、账号和授权信息只留本机，不进入 GitHub。
- **实用场景优先**：服务电商图、短视频素材、包装和海报、CAD 零件图、AI 绘画工作流、修图抠图、线稿矢量化等实际工作。
- **可复用协作**：GitHub 只保存公开安全的协议、脚本和示例，方便后续让 Jules 或其他协作者只读检查、提出改进。

核心目标有两个：一是让 AI 能协助操作复杂创作工具；二是让本地资产、账号、客户资料和商业素材始终保持可控。

## 本机路径和个人信息处理

公开仓库不保存个人安装路径、素材路径、桌面路径、微信临时文件路径、账号信息或授权信息。每台电脑通过环境变量或本地 `.env` 管理路径，状态脚本只读取这些本地配置。

| 项目 | 环境变量 |
| --- | --- |
| ComfyUI 启动脚本 | `COMFY_LAUNCHER` 或 `COMFY_START_SCRIPT` |
| ComfyUI 根目录 | `COMFY_ROOT` 或 `COMFYUI_PATH` |
| ComfyUI 输出目录 | `COMFY_OUTPUT_DIR` |
| Blender 可执行文件 | `BLENDER_EXE` |
| Blender MCP 桥目录 | `BLENDER_MCP_DIR` |
| AutoCAD 可执行文件 | `AUTOCAD_EXE` |
| Photoshop 可执行文件 | `PHOTOSHOP_EXE` |
| Illustrator 可执行文件 | `ILLUSTRATOR_EXE` |
| 下载收件箱 | `STARBRIDGE_DOWNLOAD_INBOX` |

模型、生成图、渲染输出、DWG 成品、PSD/AI 工程、临时缓存和本机路径配置不要放进 Git 工作区。

## 五条核心桥

### 1. Codex x ComfyUI：本地图像生成桥

ComfyUI 是本地生成引擎。Codex 通过 `http://127.0.0.1:8188` 调用 ComfyUI API，读取系统状态、checkpoint 列表、队列状态，提交 workflow JSON，并返回输出路径。

适合中国用户的常见场景：电商主图、国风插画、短视频封面、产品概念图、批量提示词实验、局部修复和高清放大。

已经具备：

- `examples/comfy_bridge/comfy_probe.py`：只读探针，检查 ComfyUI、显卡和 checkpoint。
- `examples/comfy_bridge/run_txt2img.py`：提交基础文生图 workflow。
- `examples/comfy_bridge/workflows/txt2img_basic_api.json`：API 格式 workflow。
- `examples/comfy_bridge/workflows/txt2img_basic_visual.json`：可在 ComfyUI 画布查看的 workflow。

### 2. Codex x Blender：三维场景桥

Blender 是三维创作引擎。Codex 可以通过 Blender Python 或 MCP 桥接层生成场景、模型、材质、灯光、相机、动画和导出文件。

适合中国用户的常见场景：产品展示台、展陈草图、空间构图、直播间/展厅概念、包装或工业设计的三维初稿。示例脚本只使用程序生成的几何体、材质和灯光，不引用私有贴图和资产库。

### 3. Codex x CAD / AutoCAD：工程制图桥

CAD 是精确制图引擎。Codex 通过本地 CAD MCP / COM 自动化层和 AutoCAD 交互，把结构化规格转换成线、圆、孔位、标注、图层和 DWG 输出。

适合中国用户的常见场景：机械零件草图、连接板、孔位图、尺寸标注、标准化图纸初稿。客户 DWG、商业图纸和授权文件只留本机，不提交到 GitHub。

已经具备：

- `cad-mcp-autocad/`：AutoCAD MCP 子项目。
- `AUTOCAD_MCP_SETUP.md`：本地安装、依赖和 Codex MCP 配置记录。
- `scripts/test_autocad_mcp.py`：MCP 连接测试脚本。
- `scripts/draw_connection_plate_from_spec.py`、`scripts/draw_reference_mechanical_part.py`：直接绘图示例。

### 4. Codex x Photoshop：修图和抠图桥

Photoshop 是修图和图层处理引擎。Codex 可以通过 Windows COM 调用 Photoshop JavaScript，完成最小接入：连接已打开的 Photoshop、创建测试文档、导出 PNG、调用主体选择并生成透明图。

适合中国用户的常见场景：电商抠图、海报素材整理、批量导出、透明 PNG、修图流程半自动化。复杂背景仍需要人工确认或二次蒙版清理。

已经具备：

- `docs/photoshop-codex-bridge.md`：Photoshop 本地桥方案、实验结论和安全边界。
- `examples/photoshop_bridge/README.md`：运行说明和边界。
- `examples/photoshop_bridge/scripts/com_probe.ps1`：创建测试文档并导出 PNG。
- `examples/photoshop_bridge/scripts/extract_subject_to_png.ps1`：输入图像转透明 PNG 的主体抠图实验。

### 5. Codex x Illustrator：矢量设计和线稿矢量化桥

Illustrator 是矢量图形和版式设计引擎。Codex 可以通过 Windows COM 调用 Illustrator JavaScript，完成最小接入：连接已打开的 Illustrator、创建测试画板、绘制基础矢量对象、调用 Image Trace，把线稿或黑白图转成 SVG / AI 矢量文件。

适合中国用户的常见场景：线稿矢量化、图标草图、包装刀版辅助图、品牌物料初稿、SVG/PDF 导出。输入图片路径和输出路径必须通过参数传入，不把微信临时路径、桌面路径或客户图稿路径写进仓库。

后续应继续把稳定动作封装成 MCP 工具，例如 `get_document_info`、`trace_image_to_vector`、`export_svg`、`export_png`。

## 扩展工具方向

星桥三联目前以 ComfyUI、Blender、CAD、Photoshop、Illustrator 为主线。后续可以在同一安全边界下扩展到更多绘画和设计工具：

| 方向 | 适合工具 | 优先级 |
| --- | --- | --- |
| 图像生成与工作流 | ComfyUI、Stable Diffusion、Flux 相关工具 | 高 |
| 三维建模与渲染 | Blender、Blender MCP | 高 |
| 工程制图 | AutoCAD、CAD-MCP | 高 |
| 修图和批处理 | Photoshop COM、Adobe UXP、Photoshop MCP | 高 |
| 矢量设计和线稿矢量化 | Illustrator COM、Image Trace、SVG/PDF 导出 | 高 |
| 设计稿协作 | Penpot MCP、Figma MCP | 中 |
| 开源画板 | Krita 及其脚本或插件 API | 待评估 |

凡是涉及登录、订阅、验证码、OAuth、浏览器授权、团队权限或账号审批的步骤，都必须由用户在官方页面手动完成。

## 安全边界

允许进入 GitHub：

- 协议文档、README、运行说明。
- 示例 Python / PowerShell 脚本和安全的 workflow JSON。
- 不含私人素材、账号信息和真实客户数据的演示配置。
- 可公开协作的任务说明和 Jules 只读任务提示。

禁止进入 GitHub：

- 密码、验证码、Cookie、token、OAuth 缓存。
- 浏览器资料、支付信息、账号风控页面内容。
- 微信临时文件、桌面路径、个人素材路径和真实输出目录。
- ComfyUI 模型、LoRA、VAE、ControlNet、生成图。
- Blender 私有 `.blend`、贴图、资产库和渲染缓存。
- CAD 客户图纸、商业 DWG、授权文件和真实项目输出。
- Photoshop 安装路径、Creative Cloud 缓存、PSD 私有工程、商业素材、源图路径和导出结果。
- Illustrator 安装路径、Creative Cloud 缓存、AI 私有工程、商业素材、源图路径和导出结果。

## 工作原则

1. **本地优先**
   生成、渲染、模型加载、CAD 自动化、修图、矢量化和资产处理优先在本机完成。

2. **中文优先**
   面向中国用户的入口说明、任务提示、风险说明和后续任务优先使用中文，降低协作和复现门槛。

3. **工具分工清楚**
   ComfyUI 管图像生成，Blender 管三维，CAD 管工程图，Photoshop 管修图，Illustrator 管矢量设计，Codex 负责连接、解释、脚本化和验证。

4. **GitHub 只保存可公开内容**
   仓库保存协议、脚本和示例，不保存模型、输出、缓存、授权文件、客户资料和私有素材。

5. **人工处理账号和授权**
   任何登录、订阅、验证码、OAuth、支付、账号审核都不交给自动化流程。

## 结论

星桥三联是一套面向中国用户的本地创作工作台组织方式：Codex 负责把需求拆成脚本和流程，GitHub 保存可公开协作的工程说明，Jules 做异步只读审查，ComfyUI 生成图像，Blender 构建三维，CAD 输出精确图纸，Photoshop 处理修图和抠图，Illustrator 处理矢量设计和导出。每条桥保持清楚边界，既提升创作效率，也让本地资产、客户资料和账号安全保持可控。
