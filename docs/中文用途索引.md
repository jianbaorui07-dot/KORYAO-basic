# 中文用途索引

这个索引用中文说明仓库里每个主要入口的用途。当前仓库只发布和 **Codex Skill + StarBridge MCP + Adobe UXP / 本地代理接入本地创意软件** 有关的公开安全内容，覆盖 ComfyUI、CAD / AutoCAD、Photoshop、Illustrator、Blender、剪映 / CapCut。这里的“接入”现在分三层：Codex skill 负责选择路线和验证命令，StarBridge MCP stdio tools 负责把已收敛的安全能力交给 AI 客户端调用，Adobe UXP / Node Proxy 负责 Photoshop 等桌面软件的可审计插件通道；这不等于所有软件都已经完整自动化。

## 状态和命令

| 原文件 | 中文用途 |
| --- | --- |
| `scripts/collect_bridge_status.py` | 读取 `examples/*_bridge/bridge_status.json`，输出 Markdown 或 JSON manifest |
| `examples/bridge_status.py` | 探测 ComfyUI / Blender / CAD / Photoshop / Illustrator / 剪映 / CapCut 本机环境 |
| `starbridge_mcp/mcp_server.py` | 真正 MCP stdio server，向 AI 客户端暴露 `tools/list` 和 `tools/call` |
| `starbridge_mcp/core/tool_registry.py` | StarBridge 能力注册表，标注每个工具的成熟度、风险和确认要求 |
| `starbridge_mcp/core/operation_context.py` | 纯内存生成可链式 operation context、状态差异、warning 脱敏和逻辑 evidence 引用 |
| `starbridge_mcp/core/queue_snapshot.py` | 默认 plan-only；显式 live probe 时读取 loopback `/queue` 并省略 workflow、history 和原始 prompt id |
| `starbridge_mcp/core/progress_monitor.py` | 默认 plan-only；显式 live connect 时有界监听直接 loopback `/ws`，输出哈希 job/node、单调进度与 stalled/终态 |
| `starbridge_mcp/core/job_snapshot.py` | 默认 plan-only；显式 live probe 时直接查询单个 loopback job，输出哈希 ID、标准化状态与终态 |
| `uxp/photoshop-bridge/` | Photoshop UXP 插件原型，服务可审计 typed handler，不保存账号或私有 PSD |
| `node_proxy/photoshop-bridge/` | Photoshop UXP / MCP 本地代理示例，连接 Node Proxy、WebSocket 和 MCP adapter |
| `scripts/security_check.py` | 检查禁止提交的资产类型、敏感路径和密钥样式文本 |
| `tests/` | 离线测试：manifest schema、operation context、脚本路径、安全检查、ComfyUI workflow 校验、CAD 解析、报告格式 |

常用命令：

```powershell
npm.cmd run status:manifest
npm.cmd run status:manifest:json
npm.cmd run status:probe:json
npm.cmd run starbridge:mcp
npm.cmd run starbridge:tools:safe
npm.cmd run comfy:probe
npm.cmd run comfy:txt2img -- --prompt "a quiet futuristic tea house" --ckpt "<checkpoint-name>"
npm.cmd run photoshop:diagnose
npm.cmd run illustrator:vectorize:offline -- --input "<input.png>" --reference-id reference
npm.cmd run security:check
npm.cmd test
```
