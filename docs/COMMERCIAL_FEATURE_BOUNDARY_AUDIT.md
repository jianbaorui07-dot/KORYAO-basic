# CreNexus 许可证与商业功能边界审计

审计基线：`503afa79c32c42b82acece7f51a12012a6709af0`  
审计日期：2026-07-17  
适用范围：公开 Community 仓库与未来独立的私有 Pro 实现

## 结论

当前版本以 CreNexus 自有许可证发布，版权人为菅宝瑞。许可证切换不追溯改变历史版本：此前已经按 MIT License 发布的提交仍受其发布时随附的 MIT 条款约束，不能通过后续产品命名把同一历史实现重新描述为 Pro 独占能力。

因此，产品中现有的“高级矢量化”定义不成立：它与已经公开的“匠心矢量”重叠。基础 Project、CreativeJob、单任务历史、基础交付和脱敏 Evidence 也属于 Community。Pro 的价值来自批量、并发、安全检查点、高级恢复/保留策略、商业规范、新的私有增强、稳定签名安装包和专业支持。

## A. 已发布的 Community 能力

下列能力继续属于 Community。当前及后续版本的使用、复制、修改和分发以对应版本随附的许可证为准；此前按 MIT License 发布的历史版本仍保留 MIT 授予的权利。

| 能力 | 公开证据 | 当前真实边界 | 商业处理 |
| --- | --- | --- | --- |
| 智能矢量 | `starbridge_mcp/vectorization/presets.py`、`engine.py`、CLI、GUI、测试与 `docs/vectorization-modes.md` | 已有公开实现；单张授权 PNG/JPEG，本机输出 SVG、预览与报告 | 保留为 Community |
| 轻量矢量 | 同一公开矢量引擎、预设、GUI 与测试 | 已有公开实现；减少颜色、碎片和节点 | 保留为 Community |
| 精确重建 | `engine.py`、`exact_pixel_vector.py`、验证器、测试与 `docs/exact-pixel-vectorization.md` | 已有公开实现；按 RGBA 像素网格确定性重建，复杂输出可能很大 | 保留为 Community |
| 匠心矢量 | `artisan.py`、公开预设、CLI、GUI、测试与 `docs/artisan-vector-mode.md` | 第一轮少锚点、角点保护、三次贝塞尔和误差门槛已公开 | 保留为 Community；不得改名后作为 Pro 独占 |

| Blender 接入 | 公开 probe、场景 plan、参考重建 plan、桥代码和测试 | 当前主要是 prototype/plan，不等于真实渲染交付 | 现有公开计划与实现继续开放；未来真实商业增强可进入 Pro |

### 不能再作为独占收费功能的项目

- 匠心、智能、轻量、精确四种现有矢量模式本身；
- 当前公开的匠心少锚点与贝塞尔算法；
- 当前公开的精确像素重建与 SVG 验证器；
- 当前公开的 Illustrator 脚本、协议、schema、UXP/Node Proxy 和已有交付路线；
- 当前公开的 Photoshop、ComfyUI、Blender、CAD/AutoCAD 探针、计划、协议、示例、桥实现和测试；
- Community 的本地运行、安全确认、路径脱敏和任务证据基础。

现有 Community 能力可以由版权人用于商业版本的组合、打包、支持和分发；对历史 MIT 版本已经授予的权利不作追溯变更。

## B. 未来新的私有 Pro 能力

未来私有 Pro 仓库只能承载在本审计基线之后新建、且未进入公开 Community 历史的实现，例如：

| Pro 能力族 | 私有实现边界 | 不能替代的事实 |
| --- | --- | --- |
| 批量队列 | 多任务排队、限流、暂停、继续、失败策略和优先级 | 不能把公开的单张矢量化改称 Pro |
| 文件夹批处理 | 用户显式选择目录后的受控批处理、范围预览和写入确认 | 不能扩大 safe roots 或扫描未授权目录 |
| 商业预设 | 新建并持续维护的行业交付预设、版本化参数与兼容策略 | 公开预设仍为 Community |
| 高级项目策略 | 跨项目检索、版本保留策略、商业交付规范 | 基础项目和单任务历史继续属于 Community |
| 高级可恢复任务 | 持久化检查点、断点续作、批量恢复验证 | 基础失败处理和单任务回滚继续属于 Community |
| 高级导出与交付 | 新建的交付打包、命名、版本、校验、客户清单和专业格式增强 | 已公开 Illustrator 路线不能被收回 |
| 新的私有算法改进 | 基线之后独立开发且未公开的算法、模型或优化 | 不能复制公开匠心实现后改名收费 |
| Adobe 商业工作流增强 | 新的批量编排、交付自动化、恢复和企业策略 | 历史 MIT 版本的 Photoshop/Illustrator 代码权利不追溯变更 |
| ComfyUI 商业工作流增强 | 新的批量编排、资产管理、恢复和商业预设 | 历史 MIT 版本的公开模板与桥权利不追溯变更 |
| Blender 商业工作流增强 | 新的真实生产编排、渲染交付、恢复和企业定制 | 历史 MIT 版本的 plan/probe 权利不追溯变更 |
| 签名离线更新包 | 私有构建、签发、验签、备份、更新与回滚实现 | 不允许静默下载、无签名安装或远程停用 |
| 专业支持与企业定制 | 人工支持、SLA、部署包、培训和定制开发 | 不改变历史版本已经授予的许可证权利 |

## 产品与 Manifest 修正规则

1. 删除以 `vectorization.advanced` 表示 Pro 独占“高级矢量化”的产品定义。
2. Pro 主能力统一为“生产级矢量工作流”。
3. Manifest 必须分别记录“公开功能已存在”和“私有增强仍在规划”，不能因为页面或锁定卡片存在就把能力标为可用。
4. Community 应明确包含四种现有矢量模式；第三方软件联动按每个公开实现的真实证据等级单独记录。
5. Pro 功能 ID 应描述新增工作流能力，例如批量队列、文件夹批处理、高级项目策略、批量恢复和商业交付规范，避免复用公开算法或 Community 基础模型名称。
6. 公共接口可以留在 Community；私有实现只能在商业构建时通过编译期边界链接，不能把完整 Pro 逻辑放进 Community 二进制后仅靠 CSS 或许可证布尔值隐藏。

## 持续审计门槛

- 每次新增 Pro 功能前，检查同名或同等实现是否已经进入公开 Git 历史。
- 商业构建固定引用经过审计的 Community commit 或版本。
- Community CI 必须证明未包含私有 Pro 源码、生产私钥或生产公钥配置。
- 所有产品页面、README、Manifest 和授权功能列表使用同一边界定义。
- 对实验、计划、mock、schema、单元测试和真实桌面验收分别标注，不互相替代。
