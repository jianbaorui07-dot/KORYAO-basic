# CreNexus Windows 第二台电脑阶段验收

日期：2026-07-22  
分支：`codex/integrated-client-alpha2`

## 客户目标

客户应当能够在 CreNexus 桌面软件内完成以下闭环：

1. 关联当前 Codex 会话，并从可见的“Codex 对话”工作区提交要求。
2. 创建项目、明确导入图片、运行矢量工作流并处理逐段确认。
3. 在交付页查看真实注册产物、哈希、质量证据和安全相对路径。
4. 从真实 PNG/JPEG 或 SVG 选择来源，选择保存路径并导出 PSD 或 AI。
5. 任何失败都返回真实错误；不覆盖源素材，不伪造格式，不使用 Image Trace 回退。

## 已通过阶段

### Windows 图片矢量化与 Pixel Vector 客户闭环

- 精确基线支持 512、1024、1600、2048 或原始最长边。
- SVG 安全上限支持 64、128 或 256 MB；绝对硬上限仍为 256 MB。
- 默认绘制策略改为智能矢量，工匠曲线保留给 Logo 与扁平几何。
- 失败任务可以返回工作流调整参数后重新建立。
- 一张合法客户测试图片从 CreNexus 客户界面完成真实桌面任务，依次通过写入确认并到达 100%。
- 同一项目累计显示 30 个注册产物和 3 份证据；本次任务新增 10 个真实产物和 1 份证据。
- 精确工作副本为 768×1024，像素验证零差异，SVG 约 34.7 MB。
- 客户界面将该确定性路径显示为“像素矢量”，内部 `exact` 标识保持不变以兼容旧任务。
- 智能矢量为 1086×1448，SVG 约 3.7 MB，不含嵌入位图或外部引用。
- 桌面交付的 Pixel Vector SVG 为 34,729,131 字节，SHA-256 为 `7E4751D6B90483FD4AD57ED0A94C96B379B6B48769E6F484183595E42B51F9D8`；无位图、脚本或外部引用。

### Codex 对话入口

- 左侧导航新增“Codex 对话”。
- 支持可选项目上下文和用户要求输入。
- 提供“用户视角”“制作者视角”“左右迭代”模板。
- 使用 Codex 深链和已配对的 CreNexus MCP 交接真实任务。
- 不读取 Codex 登录凭据，不把本地占位文字伪装成模型回复。
- 未采用仍属于实验接口的 Codex app-server 作为客户稳定依赖。
- 实机确认 Codex 不会在运行中热加载首次安装的 MCP 连接器；界面现在明确要求完全退出并重新打开 Codex，再在新任务中完成配对。

### PSD / AI 客户交付闭环

- AI 只接受当前项目中的真实 SVG 产物。
- PSD 只接受当前项目中的真实 PNG/JPEG 产物。
- 用户必须明确确认，并通过系统保存窗口选择路径。
- Photoshop / Illustrator 使用固定 COM + ExtendScript 保存，不接受任意脚本输入。
- Adobe 保存后必须原生重开验证。
- PSD 必须通过 `8BPS` 文件签名；AI 必须通过 PDF-compatible 或 Adobe PostScript 签名。
- Adobe 先写应用私有暂存目录，验证后使用 `create_new` 发布到客户路径。
- 目标已存在或出现并发抢占时拒绝覆盖；测试确认客户原文件字节不变。
- 成功导出后保存文件名、来源文件名、大小、SHA-256、时间和原生重开结果。
- 导出历史在重启后仍可按项目读取，最多显示最近 100 条。
- 审计 JSON 明确不保存客户目标文件的绝对路径；历史写入失败不会否定已经验证并发布的客户文件。
- Illustrator 2026 已从约 33.9 MB 的真实 Pixel Vector SVG 生成并原生重开 `中央红鲤_像素矢量_alpha2.ai`；文件为 17,485,806 字节，SHA-256 为 `9F864573B89D5E662348C4F36A5E08792061E29ACD65B3705E2B3F8EA8BAB2BE`。
- Photoshop 2026 已从注册预览 PNG 生成并原生重开 `中央红鲤_像素矢量_alpha2.psd`；文件为 4,754,484 字节，SHA-256 为 `4A9E49A82D222F01F158470AFAF78B22343BBD1BA0670E5537306E76A575B88D`。
- AI 通过 `%PDF-1.6` 签名检查，PSD 通过 `8BPS` 签名检查；两条持久化导出回执均显示 `nativeReopenValidated=true` 且未保存绝对目标路径。
