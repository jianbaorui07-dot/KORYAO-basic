# WF-0009 Art Gap Report

Status: blocked / staged / scene_builder_ready.

This report does not claim the UE scene is complete. It records what is ready and what must still be created inside Unreal Editor when free memory reaches the required threshold.

## Current proxy plan

- Temple structure: editable primitive columns, beams, roof, eaves, plinths, and shrine marker will be created in UE by `Content/Python/worldforge/worldforge_scene_builder.py`.
- Robot: editable primitive body proxy will stand in for a high quality robot mesh. Replacement target: `BP_WF0009_RobotScout` components and material slots.
- Mountains: editable primitive silhouette blocks will stand in for sculpted terrain or licensed mountain meshes. Replacement target: actors labeled `WF0009_MountainBackdrop_*`.
- Weather: low cost `ExponentialHeightFog`, cold directional light, and skylight will stand in for advanced Niagara snow and volumetric polish.
- Interaction: `WF0009_TempleApproach_TriggerBox_PendingPIE` plus `WF0009_RobotScout_ChestLight_FeedbackTarget` will mark the first feedback loop. PIE proof is still required.

## Replacement safety

Future higher quality meshes can replace primitive components inside the same Blueprint assets without changing the map contract, trigger ownership, WorldSpec path, or validation rules.

## Block reason

Free physical memory was about 8.52 GB before launch, but fell to 6.09 GB inside Unreal Editor. WorldForge Resource Governor requires at least 7.0 GB inside the editor before asset creation, 8.0 GB before PIE verification, and 10.0 GB before cinematic render.
