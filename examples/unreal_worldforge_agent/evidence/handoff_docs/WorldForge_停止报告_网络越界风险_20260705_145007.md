# WorldForge 停止报告：网络越界风险

生成时间：2026-07-05 14:50:07 +08:00

## 停止原因

UE 日志在关闭阶段记录 FHttpManager::Flush，目标为 https://datarouter.ol.epicgames.com/datarouter/api/v1/public/data?...，判断为 UE/Epic telemetry 类外部 HTTP 记录。

该行为不是 WorldForge 脚本主动发起，也未下载外部资产；但它已经超出“网络仅本机”的边界。因此按强制停止规则，停止后续 Remote Control 和 MCP 接入。

## 停止时状态

- UE 进程：已正常退出，无 UnrealEditor.exe / UnrealEditor-Cmd.exe 残留。
- 端口状态：30010/30020/8000/6666/30000 均为空闲；未发现残留监听。
- 原始项目8：未修改；与只读备份哈希差异数 0。
- 已完成内容：Content/WorldForge 下的基础材料、蓝图壳、WBP、M_WorldForgeLab、M_WorldForgeBlockoutSandbox、未来城市 Blockout、20 秒 PIE Smoke Test。
- 未继续内容：Remote Control 服务未启动；Remote Control Preset 未创建；MCP 桥接未创建；项目级 .codex 配置未写入。

## 下一步建议

1. 先在 UE 项目或启动参数层面关闭 Epic telemetry / DataRouter 外发能力，并重新做一次只读启动验证。
2. 重新验证日志中没有外部 URL、0.0.0.0、组播、局域网地址和异常端口监听。
3. 只有验证通过后，再进入 Remote Control 本机绑定和 MCP stdio 桥接。
